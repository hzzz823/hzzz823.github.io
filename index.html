  1. 要做往dataFlow中添加input，output，和各种actions
  2. input没有别的只需要可以给他配置数据来源哪个table。（& 预览数据功能）
  3. output没有别的只是作为标记数据最终结果（出口）。（数据转换结束）

所以要做的就是： input + action = output
只不过一个dataFlow中可以有不止一条i + a = o
只不过一条i + a = o中，input可以是多个，action可以是多个，output一定是一个。

  1. 关于实现，每条inputs + actions = output，input作为起点，output作为终点，中间每个节点有输入（actionIn）和输出（actionOut）和各自的转换行为。（比如为输入添加一列常数列），所以actionIn + action提供的转换 = actionOut，而actionIn = 上一节点的actionOut。

上边是比较细节的。 或者说只是关于具体实现的方案， 而还有很多比如校验等功能模块。

总之，需要先从整个大
功能，来分析一下，要做什么。 先来想想大积木应该是什么样。

要做的事情分一下应该。
比如要有一个sql嵌套这个API。 或者是用view。
比如中间表不用存，只要最终结果表。
比如要考虑结果怎么存。

API：

  1. query resources type list
  2. query resources owner list
  3. queryAllTables by type & owner
  4. 选择table要返回columns:[type:{columnType}, name:”columnName”]
  5. update

Plan:

  1. 原来认为的input/output/action 其实都是action。 只是类别不同。
  2. 不同的trans action是不同type的action。 可以看保存的actions.json。
  3. 错误用configuration error。 可以看保存的configurationErrors.json。
  4. 每次改动都更新，并返回最新。可以看request.json和response.json。
  5. output属于一种dataSource。

hive install

1 install hive

  1. brew install hive
  2. brew install mysql
  3. add /usr/local/Cellar/hive/2.1.0/libexec/lib in ~/.zshrc
  4. mysql -uroot
  5. CERATE USER 'hiveuser'@'localhost' IDENTIFIED BY 'password'
  6. GRANT ALL PRIVILEGES ON *.* TO 'hiveuser'@'localhost' IDENTIFIED BY 'password'
  7. download msyqlConnector curl -L 'http://www.mysql.com/get/Downloads/Connector-J/mysql-connector-java-5.1.40.tar.gz/from/http://mysql.he.net/' | tar xz
  8. cp mysql-connector-java-5.1.40/mysql-connector-java-5.1.40-bin.jar /usr/local/Cellar/hive/2.1.0/libexec/lib
  9. cd $HIVE_HOME/conf
  10. cp hive-default.xml.template hive-site.xml
  11. vi hive-site.xml
  12. hive

JUnit杂记

@ignore

Sometimes you want to temporarily disable a test or a group of tests. Methods annotated with Test that are also annotated with @Ignore will not be executed as tests. Also, you can annotate a class containing test methods with @Ignore and none of the containing tests will be executed. Native JUnit 4 test runners should report the number of ignored tests along with the number of tests that ran and the number of tests that failed.
