Posted on 2016-10-16 | In 程序员

这一周接触了很多东西，整理一下。

重建了一个博客，也不知道为什么，折腾了好久。可能是没设CNAME吧。
重新理一下建博客的过程，其实很简单。

1. hexo + github

我是mac系统，然后我的域名是m.xydayer.me,github名字是hzzz823，过程中需要换自己的。

  1. github创建repository，命名hzzz823.github.io
  2. 装hexo，brew cask install hexo
  3. 在你想要的目录下，hexo init blog，blog是文件夹名，自己起
  4. 在blog目录下，hexo init,npm install

  5. vi _config.yml，在最后边，修改deploy配置（注意冒号后边有空格。）

  6. npm install hexo-deployer-git --save 否则git提交会失败。

  7. 在blog/source下创建一个文件’CNAME’里边存域名，我的是’m.xydyaer.me’
  8. 在blog目录下，hexo generate,hexo deploy。然后访问你的域名就可以看到博客了。

大概就这个过程，工具方面可能会缺node，brew,git…的话装一下。配置方面github没配置过的话也要配一下，不然hexo deploy这一步应该上传会失败。

然后想换主题的话去网上搜，搜好之后下载到blog/themes下，再修改blog/_config.yml中的theme，修改为你刚下的主题名称。

另外一些blog的基础设置，标题之类，也全在blog/_config.yml

2. dropwizard-helloworld

dropwizard是一个java框架，简介看官网这段介绍说的很清楚

Dropwizard is a Java framework for developing ops-friendly, high-performance, RESTful web services.

Dropwizard pulls together stable, mature libraries from the Java ecosystem into a simple, light-weight package that lets you focus on getting things done.
Dropwizard has out-of-the-box support for sophisticated configuration, application metrics, logging, operational tools, and much more, allowing you and your team to ship a production-quality web service in the shortest time possible.

  1. create config.class
  2. create a config file: config.yml (放在项目根目录下)
  3. create application.class extends Application

  4. create a resource class

  5. register resources in application.class

主要部件就是config.class,config.yml,application.class,然后resource中就是业务方面了，在application中register即可。
